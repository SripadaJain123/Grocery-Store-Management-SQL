# Grocery Store Management SQL Project

## Project Overview
This project is a SQL-based Grocery Store Management System designed to efficiently manage customers, suppliers, employees, products, categories, orders, and sales transactions.

The project demonstrates relational database design, SQL querying, business analysis, and data-driven insights using structured datasets.

---

## Objectives
- Design a centralized relational database system
- Manage grocery store operations efficiently
- Track customer purchases and sales trends
- Analyze supplier and employee performance
- Generate business insights using SQL queries
- Improve operational efficiency and inventory management

---

## Technologies Used
- SQL
- MySQL
- Relational Database Management System (RDBMS)
- CSV Datasets

---

## Database Tables
- Categories

## SQL Analysis Performed

### Customer Insights
- Unique customers count
- Customers with highest number of orders
- Top customers by total purchase amount
- Average customer spending

### Product Performance
- Products available in each category
- Best-selling products
- Revenue generated by products
- Average product pricing by category

### Sales & Order Trends
- Total orders placed
- Average order value
- Peak sales dates
- Weekday vs weekend sales analysis

### Supplier Contribution
- Supplier product count
- Supplier revenue contribution
- Average product pricing by suppliers

### Employee Performance
- Orders processed by employees
- Total sales value handled
- Average order value per employee

---

## Key Insights
- A small group of customers contributes major revenue.
- Top-selling products require consistent stock availability.
- Weekend and month-end sales are significantly higher.
- Some categories need promotional strategies.
- Supplier diversification is necessary for operational stability.
